🧹 [code health] Add logging to empty catch blocks in ManagedVenvHostManager

🎯 What: The code health issue addressed
Replaced empty `catch` blocks with explicit exception catching and warnings inside `ManagedVenvHostManager.Dispose()`.

💡 Why: How this improves maintainability
Swallowing exceptions silently can hide bugs and mask the root cause when trying to terminate stray GPU host processes. Logging these failures explicitly makes debugging easier and prevents silent process leaks or recovery failures.

✅ Verification: How you confirmed the change is safe
Ran the `dotnet test` suite, verifying `ManagedVenvHostManagerTests` successfully passed. Added no side effects to the actual teardown logic.

✨ Result: The improvement achieved
Maintains the resilient dispose architecture while surfacing hidden errors instead of swallowing them.
